# دليل مدير المنصة (Platform Admin)

## نظرة عامة
هذا الدليل يشرح لمدير المنصة كيفية إعداد النظام متعدد المدارس خطوة بخطوة، وإدارة الصلاحيات والوحدات، ومراقبة الأداء.

## البدء السريع
1. تسجيل الدخول بحساب المدير العام.
2. إنشاء مدرسة جديدة: من صفحة "المدارس" اضغط "إضافة مدرسة" وحدد بياناتها.
3. تفعيل الاشتراك/الخطة: اختر الخطة المناسبة واضبط مدة التجربة إن لزم.
4. إنشاء مدير مدرسة: أدخل بيانات المسؤول وسيُنشأ حسابه تلقائيًا.

## إعدادات عامة
- تفعيل/تعطيل الوحدات لكل مدرسة (المالية، النقل، الرسائل، التحليلات).
- ضبط سياسات الأمان: MFA، حدود الـ IP، عمر الجلسة.
- إدارة الترخيص: حفظ `LICENSE_KEY` وضبط الوحدات المفعلة.

## إدارة المستخدمين والصلاحيات
- الأدوار: Super Admin، School Admin، Teacher، Parent.
- ضبط صلاحيات دقيقة للمستخدمين عبر قائمة الأذونات (permissions) لكل مدرسة.

## مراقبة الأداء
- استخدام لوحة التحليلات العامة لمراجعة الاستخدام والطلبات.
- تأكيد الفهارس عبر صفحة الصحة `/api/health`.

## استكشاف الأخطاء
- راجع السجلات (Logs) وابحث عن رسائل `access_denied_*` و`SERVER_ERROR`.
- تأكد من صحة متغيرات البيئة (JWT/Redis/DB).
